What is HTML?

Back when the world was a safer place for us Weasels, the web was a much simpler place. Before CSS makes it pretty and JavaScript makes it dance, the early web was just HTML and vibes, man! It was but a collection of documents linked together. Yes it was ugly, yes it was chaotic, yes it was raw. It was the wild west of the internet, and everyone loved it.

Yahoo looked like a spreadsheet, GeoCities looked like a ransom note, and nobody cared because it was all magic nonetheless.

HTML (HyperText Markup Language) uses elements to describe the structure and meaning of content. The browser reads your HTML and renders it as a visual page. We are talking about the main building block, the skeleton, of the web page here. Don't worry if it looks simple now.. you'll be building amazing things in no time!

Cool, right..!? Ready to dive in?

Basic HTML structure

Every HTML file starts with <!DOCTYPE html> a one-liner that tells the browser to use modern HTML5 rules.
It's like a letter. The <head> is the envelope; nobody reads it but it tells the post office what to do. The <body> is the letter itself. That's where all the show goes on! 

<!-- A minimal HTML5 document -->
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title>My First Page</title>
  </head>
  <body>
    <h1>Hello, world!</h1>
    <p>This is a paragraph.</p>
  </body>
</html>

See what a basic HTML document looks like in the browser.

Common HTML elements

HTML has over 100 elements, but don't worry..you will only use a small set of them most of the time. Here are the most important ones: 

<!-- Headings: h1 is largest, h6 is smallest -->
<h1>Main heading</h1>
<h2>Sub-heading</h2>

<!-- Paragraph -->
<p>This is a paragraph of text.</p>

<!-- Hyperlink: href sets the destination -->
<a href="quiz.html">Go to quiz</a>

<!-- Image: src is the file path, alt is for screen readers -->
<img src="assets/img/photo.jpg" alt="My photo" />

<!-- Unordered list -->
<ul>
  <li>First item</li>
  <li>Second item</li>
</ul>

See what these common elements look like rendered in the browser.

Semantic HTML

Semantic elements describe the purpose of the content, not just how it looks. This helps search engines and screen readers understand your page. Use these instead of generic <div> elements wherever possible. 

<header>  <!-- top of the page -->
<nav>     <!-- navigation links -->
<main>    <!-- main page content -->
<section> <!-- a standalone section -->
<article> <!-- self-contained content -->
<footer>  <!-- bottom of the page -->

HTML forms

Forms are how the web collects information from users. Every login page, search box, and checkout form you've ever used is an HTML <form> element under the hood. Inside a form, you add <input> elements to create the actual fields. 

<!-- A simple login form -->
<form action="/login" method="post">

  <label for="email">Email</label>
  <input type="email" id="email" placeholder="[email protected]" required />

  <label for="password">Password</label>
  <input type="password" id="password" placeholder="••••••••" />

  <input type="submit" value="Log in" />

</form>

The type attribute on an input controls how it renders and what it accepts — email validates email format, password hides the text, required prevents submission if the field is empty.

HTML tables

Tables organise data into rows and columns. They get a bad reputation because developers used to abuse them for page layout. Yeah, don't do that. Tables are for tabular data only. Use CSS for layout. 

<table>
  <tr>                          <!-- table row -->
    <th>Language</th>           <!-- header cell -->
    <th>Created</th>
    <th>Used for</th>
  </tr>
  <tr>
    <td>HTML</td>               <!-- data cell -->
    <td>1991</td>
    <td>Structure</td>
  </tr>
  <tr>
    <td>CSS</td>
    <td>1996</td>
    <td>Styling</td>
  </tr>
  <tr>
    <td>JavaScript</td>
    <td>1995</td>
    <td>Behaviour</td>
  </tr>
</table>

HTML best practices

Writing valid HTML is not just about making the browser happy.. it is about writing code that is accessible, maintainable, and future-proof. Here are the rules every good HTML developer follows. 

<!-- 1. Use semantic elements over generic divs -->
<nav>...</nav>        <!-- good -->
<div id="nav">...</div> <!-- avoid -->

<!-- 2. Always include alt text on images -->
<img src="weasel.png" alt="WeaselCodes mascot" /> <!-- good -->
<img src="weasel.png" />                       <!-- avoid -->

<!-- 3. Always close your tags -->
<p>This is correct.</p>   <!-- good -->
<p>This will cause bugs.  <!-- avoid -->

<!-- 4. Use lowercase for all tag names and attributes -->
<input type="text" />   <!-- good -->
<INPUT TYPE="TEXT" />   <!-- avoid -->

<!-- 5. Declare the language on the html element -->
<html lang="en">   <!-- helps screen readers and search engines -->

Always validate your HTML at validator.w3.org before submitting. A single unclosed tag can break an entire page layout.

What is CSS?

CSS (Cascading Style Sheets) is the reason some websites look stunning while others look like they were designed in 2001 by someone's dad. CSS controls how HTML elements look. HTML built the house. Now CSS decides whether it looks like a mansion or a disaster. Colour, fonts, spacing, layout. CSS is the paint and brush of the web.

Ready to see magic?

This live CSS editor allows you to see the effects of CSS changes in real time.
Adjust the background colour, text colour, font size, and border radius using the controls below.

One stylesheet to rule them all.


Fellas, today is the day we give CSS the respect it deserves. Let's dive into the world of selectors, properties, and the box model to see how CSS turns plain HTML into a work of art.

CSS syntax

Every CSS rule follows the same pattern. The selector says what to style; the declarations say how to style it. 

/* selector { property: value; } */

h1 {
  color: #00BFFF;
  font-size: 2rem;
  text-align: center;
}

p {
  color: #8b95a8;
  line-height: 1.7;
  margin-bottom: 1rem;
}

Selectors

CSS selectors let you target elements in different ways. The three most common are element selectors, class selectors, and ID selectors. 

/* Element selector — targets ALL paragraphs */
p {
  color: white;
}

/* Class selector — targets elements with class="highlight" */
.highlight {
  background-color: yellow;
}

/* ID selector — targets the element with id="logo" */
#logo {
  width: 120px;
}

/* Descendant selector — targets <a> inside <nav> only */
nav a {
  text-decoration: none;
}

The box model

Every HTML element is a rectangular box. The CSS box model describes the layers of that box from the inside out: content, padding, border, and margin. Understanding this is essential for controlling layout and spacing. 

.box {
  width: 300px;       /* content width */
  padding: 16px;     /* space inside the border */
  border: 2px solid #00BFFF; /* visible border */
  margin: 24px;      /* space outside the border */
}

Responsive design & CSS frameworks

Writing responsive CSS from scratch for every screen size is tedious and error-prone. This is where CSS frameworks come in. A framework is a pre-written collection of CSS (and sometimes JavaScript) that gives you a solid foundation to build on so that you can focus on design instead of mechanics.

Bootstrap is the most popular CSS framework in the world. It gives you a 12-column grid system, responsive breakpoints, and ready-made components like buttons, navbars, and cards — all working out of the box.

Loading Bootstrap via CDN

The right way to use Bootstrap is to load it dynamically from a Content Delivery Network (CDN) instead of downloading and storing it locally. A CDN serves the file from a server close to the user, which is faster, and means your project folder stays clean. 

<!-- Paste this in your <head> to load Bootstrap CSS -->
<link
  rel="stylesheet"
  href="https://cdn.jsdelivr.net/npm/[email protected]/dist/css/bootstrap.min.css"
/>

<!-- Paste this before your closing </body> to load Bootstrap JS -->
<script
  src="https://cdn.jsdelivr.net/npm/[email protected]/dist/js/bootstrap.bundle.min.js">
</script>

The 12-column grid

Bootstrap divides every row into 12 columns. You control how many columns an element spans at each screen size using classes like col-md-6 (half width on medium screens and above). On smaller screens, columns automatically stack vertically. 

<!-- Two columns side by side on medium+ screens,
     stacked on mobile -->
<div class="row">
  <div class="col-md-6">Left column</div>
  <div class="col-md-6">Right column</div>
</div>

<!-- Three equal columns -->
<div class="row">
  <div class="col-md-4">Column 1</div>
  <div class="col-md-4">Column 2</div>
  <div class="col-md-4">Column 3</div>
</div>

Custom CSS always wins

Bootstrap is a starting point, not the finish line. The key to making your site look unique is layering your own custom CSS on top of Bootstrap. The trick is to load your stylesheet after Bootstrap's, that way your rules override Bootstrap's defaults wherever they conflict. 

<!-- Bootstrap first -->
<link rel="stylesheet" href="bootstrap.min.css" />

<!-- Your custom CSS second — overrides Bootstrap -->
<link rel="stylesheet" href="css/style.css" />

This is exactly how WeaselCodes is built. Bootstrap handles the responsive grid and base styles, while style.css handles the theme, accents, custom fonts, and everything else that makes this site look awe-to-the-some, baby!

Colours & typography

CSS gives you three ways to specify colour, and full control over how text looks. These two things alone account for most of the visual personality of a website. 

/* Three ways to write the same colour */
h1 { color: cyan; }          /* named colour */
h1 { color: #00BFFF; }       /* hex code */
h1 { color: rgb(0,191,255); } /* RGB values */

/* Typography properties */
p {
  font-family: 'Courier New', monospace;
  font-size:   1rem;      /* relative to root font size */
  font-weight: 700;       /* 400 = normal, 700 = bold */
  line-height: 1.7;       /* spacing between lines */
  text-align:  center;
  letter-spacing: 0.05em;
}

Pseudo-classes

Pseudo-classes let you style elements based on their state — what the user is doing with them right now. The most common one is :hover, which fires when the mouse moves over an element. You've already seen this working on the tab buttons above. 

/* Style a link normally */
a {
  color: #00BFFF;
  text-decoration: none;
}

/* Change it when the mouse hovers over it */
a:hover {
  color: #ffffff;
  text-decoration: underline;
}

/* Style an input when it is focused (clicked into) */
input:focus {
  border-color: #00BFFF;
  outline: none;
}

/* Style every other table row */
tr:nth-child(even) {
  background-color: #1a2235;
}

Other useful pseudo-classes: :visited for links already clicked, :first-child for the first element in a group, and :disabled for disabled form inputs.

CSS best practices


CSS is easy to write badly. It is easy to write rules that conflict with each other, that break on some screens but not others, or that become impossible to maintain as the project grows. These habits keep your stylesheets clean. 

/* 1. Use external stylesheets — never inline styles */
/* Good — in style.css */
.btn { background: #00BFFF; }

/* Avoid — inline style on the element */
<button class="accent-bg"> ...too hard to maintain

/* 2. Use CSS variables for colours and repeated values */
:root {
  --accent: #00BFFF;   /* change once, updates everywhere */
}
.btn { background: var(--accent); }

/* 3. Use class selectors — avoid ID selectors for styling */
.card { padding: 24px; }  /* reusable on many elements */
#card { padding: 24px; }  /* only works on one element */

/* 4. Load your CSS after Bootstrap so your rules win */
<link href="bootstrap.css" />  <!-- first -->
<link href="style.css" />      <!-- second — overrides Bootstrap -->

/* 5. Use a mobile-first approach */
.container { width: 100%; }          /* mobile default */
@media (min-width: 768px) {
  .container { max-width: 1200px; } /* wider screens */
}

Validate your CSS at jigsaw.w3.org/css-validator. Clean, organised CSS is one of the first things a developer notices when reading someone else's code.

What is JavaScript?

God created the universe in 7 days.
JavaScript was created in 10 days.
One of these things has fewer bugs.

JavaScript is the programming language that runs in the browser. It adds behaviour. It responds to user actions, updating content dynamically, and communicating with servers.

Fair warning: JavaScript will frustrate you, confuse you, and occasionally make you question your life choices. It will also make you feel like a wizard the first time you write code that actually does something. Worth it. Trust the weasel.

Variables and data types

Variables store values. Use const for values that won't change, and let for values that will. JavaScript has several data types: strings, numbers, booleans, arrays, and objects. 

// Declaring variables
const siteName = "WeaselCodes";  // string — won't change
let score = 0;                  // number — will change
let passed = false;             // boolean

// Array — a list of values
const topics = ["HTML", "CSS", "JavaScript"];

// Object — key/value pairs
const user = {
  name: "Alex",
  score: 8,
  passed: true
};

Functions

Functions are reusable blocks of code. You define a function once and can call it as many times as you need. Functions can accept parameters (inputs) and return a value (output). 

// Function declaration
function greet(name) {
  return "Hello, " + name + "!";
}

// Calling the function
const message = greet("Alex");
console.log(message); // "Hello, Alex!"

// Arrow function (shorter syntax, same result)
const greet2 = (name) => "Hello, " + name + "!";

DOM manipulation

The DOM is JavaScript's view of the HTML page. You can use document.getElementById() to find an element, then read or change its content, style, or attributes. 

// Find an element by its id
const heading = document.getElementById("main-title");

// Change its text content
heading.textContent = "New title!";

// Change its style
heading.style.color = "#00BFFF";

// Find elements by class name
const cards = document.querySelectorAll(".card");

Events

Events let your code respond to user actions like clicks, key presses, and form submissions. You attach an event listener to an element and provide a function to run when the event fires. 

// Find the button
const btn = document.getElementById("my-btn");

// Listen for a click event
btn.addEventListener("click", function() {
  alert("Button was clicked!");
});

// Arrow function version
btn.addEventListener("click", () => {
  alert("Button was clicked!");
});

The Browser Object Model (BOM)

The DOM is just one part of a bigger picture. The Browser Object Model (BOM) is JavaScript's access to the entire browser environment. Not just the page content, but the window itself, the URL, the history, and even local storage. The root of the BOM is the window object, which represents the browser tab. 

// The window object — the global scope in browsers
console.log(window.innerWidth);   // browser window width
console.log(window.innerHeight);  // browser window height

// location — read and change the current URL
console.log(window.location.href);        // current page URL
window.location.href = "quiz.html";       // navigate to quiz

// localStorage — store data in the browser permanently
localStorage.setItem("score", "8");       // save a value
const score = localStorage.getItem("score"); // retrieve it
localStorage.removeItem("score");         // delete it

localStorage survives page reloads and even closing the browser. This is exactly what the quiz page uses to remember your previous attempts.

jQuery: JavaScript, but friendlier

Vanilla JavaScript DOM manipulation can be verbose. jQuery is a library that wraps all of that into shorter, cleaner syntax. Released in 2006, it became the most widely used JavaScript library in the world. Everything starts with the $ selector. 

// Vanilla JS vs jQuery — same result, less typing

// Vanilla JS
document.getElementById("box").style.display = "none";

// jQuery equivalent
$("#box").hide();

// Wait for the page to fully load before running code
$(document).ready(function() {

  // Click event on a button
  $("button").click(function() {
    $("#message").text("Button clicked!");
    $("#message").css("color", "#00BFFF");
  });

});

jQuery is loaded via CDN just like Bootstrap. We use it in the quiz page for AJAX requests — fetching the questions file and the reward API without reloading the page.

JavaScript best practices

JavaScript is a forgiving language (nope, not a good thing). It will let you do a lot of things that will cause you pain later. These rules exist because experienced developers learned them the hard way so you don't have to. 

// 1. Always use const or let — never var
const name = "Nevis";  // value won't change — use const
let score = 0;        // value will change — use let
var old = "avoid";    // var has confusing scoping rules

// 2. Always use strict equality ===, not ==
0 == ""   // true  — dangerous! JS converts types
0 === ""  // false — correct, different types

// 3. Keep JavaScript in external files
// In your HTML:
<script src="js/quiz.js"></script>  // good
<script> // hundreds of lines... </script> // hard to maintain

// 4. Comment your code — explain the why, not the what
let score = 0; // bad comment: "sets score to 0"
let score = 0; // good comment: "tracks running total for quiz"

// 5. Always validate user input before using it
const input = document.getElementById("age").value;
if (!input) {
  alert("Please enter your age.");
  return; // stop here — never trust empty input
}

The most important rule: write code as if the person who maintains it after you is a dangerous psychopath who knows where you live. Comment everything. Name things clearly. Keep functions small.
